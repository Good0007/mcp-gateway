# MCP Agent 产品路线图

> 最后更新：2026-02-16

## 🎯 项目愿景

打造一个功能强大、易于使用的 MCP 服务聚合平台，让开发者和用户能够轻松集成和管理多个 MCP 服务。

---

## ✅ 已完成功能

### v0.1.0 - 核心功能 (2026-02-14 完成)

- ✅ **多适配器支持** - stdio、embedded、sse、http
- ✅ **服务生命周期管理** - 启动、停止、重启、健康检查
- ✅ **工具聚合** - 自动收集所有服务的工具列表
- ✅ **小智 WebSocket 连接** - 自动重连、心跳保活
- ✅ **配置热重载** - 文件监听、动态应用

### v0.2.0 - Web 管理界面 (2026-02-15 完成)

- ✅ **服务管理页面** - 可视化添加、编辑、删除服务
- ✅ **工具列表** - 查看所有可用工具
- ✅ **日志查看** - 实时服务日志
- ✅ **环境检测** - Node/Python/Rust/Java/Go 环境管理
- ✅ **配置导入** - 支持导入 Claude Desktop/VS Code 配置

### v0.3.0 - MCP 代理服务器 (2026-02-16 完成)

- ✅ **双传输协议** - Streamable HTTP + Legacy SSE
- ✅ **VS Code 集成** - `.vscode/mcp.json` 配置
- ✅ **Claude Desktop 集成** - 配置文件支持
- ✅ **Token 认证** - Bearer Token 保护代理端点
- ✅ **代理管理界面** - Token 生成、状态监控
- ✅ **类型定义统一** - shared 包单一数据源
- ✅ **Monorepo 优化** - 构建流程、依赖管理

## 💡 待讨论的想法

以下功能尚在评估阶段，欢迎反馈：

### 🤔 服务编排
- **问题**: 当前服务独立运行，能否支持服务间协作？
- **方案**: 引入 DAG 工作流，支持服务依赖和数据流转
- **挑战**: 复杂度增加，可能超出项目范围

### 🤔 多租户支持
- **问题**: 能否支持多个团队共享一个实例？
- **方案**: 租户隔离、资源配额、计费系统
- **挑战**: 需要重新设计权限和存储

### 🤔 GraphQL API
- **问题**: REST API 是否足够灵活？
- **方案**: 提供 GraphQL 端点，支持复杂查询
- **挑战**: 维护两套 API，增加开发成本

### 🤔 桌面应用
- **问题**: Web 界面是否需要原生桌面版？
- **方案**: Electron 打包，提供更好的系统集成
- **挑战**: 增加维护负担，Web 版已足够
